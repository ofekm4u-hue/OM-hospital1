# OM Hospital — סימולטור חדר מיון (ED/EMR) בעברית

אפליקציית web עצמאית (RTL, עברית) המדמה מערכת ניהול חדר מיון מקצועית של בית
החולים **OM Hospital**: התחברות מבוססת-תפקיד עם אימות דו-שלבי, קבלה וטריאז',
תיק קליני, הזמנת בדיקות (CPOE) עם תוצאות מדומות, מרשמים עם נעילת אלרגיה, לוח
בקרה בזמן אמת, ומחולל מכתב שחרור רשמי מוכן להדפסה.

האפליקציה היא **PWA** ניתנת להתקנה ועובדת גם ללא רשת (offline). כל הנתונים
נשמרים מקומית בדפדפן (`localStorage`) — אין שרת חיצוני. **המערכת לסימולציה
ולמידה בלבד ואינה מיועדת לשימוש קליני אמיתי.**

## הפעלה

```bash
# מהשורש של המאגר:
python3 -m http.server 8000
# פתחו: http://localhost:8000/hospital/index.html
```

להתקנה כאפליקציה — פתחו בדפדפן ובחרו "התקן אפליקציה" / "הוסף למסך הבית".

## חשבונות התחברות (דמו)

המערכת מתחילה ללא מטופלים, עם חשבונות צוות מובנים. בהתחברות מוצג **קוד אימות
דו-שלבי (2FA) על המסך** לצורך הסימולציה.

| תפקיד | שם משתמש | סיסמה |
|-------|----------|-------|
| רופא (ד"ר אופק משאל) — גם מנהל מערכת | `ofek` | `Ofek2026!` |
| רופאה (ד"ר דנה לוי) | `dana` | `Dana2026!` |
| אחות (מירב כהן) | `merav` | `Merav2026!` |
| פקיד קבלה (יוסי אברהם) | `yossi` | `Yossi2026!` |

## תתי-המערכות

1. **ניהול משתמשים והרשאות (IAM)** — התחברות + 2FA מדומה, שלוש רמות הרשאה:
   פקיד קבלה (דמוגרפיה + טופס 17, חסום מקליני), אח/ות (מדדים, מתן תרופות, טריאז'),
   רופא/ה (אנמנזה, בדיקות, מרשמים, שחרור). מנהל מערכת — גישה מלאה + ניהול משתמשים.
2. **קבלה וטריאז'** — קליטת מטופל, מדדים חיוניים, וחישוב אוטומטי של צבע דחיפות
   (אדום/צהוב/ירוק) ורמת ESI 1–5.
3. **תיק קליני ואנמנזה** — תלונה, HPI, רקע, ו**התראת אלרגיה אדומה בולטת**.
4. **הפניות ולוגיסטיקה (CPOE)** — בדיקות מעבדה ודימות עם אינדיקציה קלינית,
   וסימולציית תוצאות (ערכי מעבדה עם סימון חריגות + פענוח רדיולוג).
5. **מרשמים וניהול תרופות (e-Prescribing & MAR)** — אינדקס תרופות, מינון/תדירות/
   דרך מתן, **נעילת אלרגיה** עם תיעוד חריגה, וסימון מתן בפועל.
6. **לוח בקרה (ED Dashboard)** — טבלת מיון דינמית, ניהול מיטות, וניווט מהיר לתיק.
7. **מחולל מסמכים** — מכתב שחרור רשמי בתבנית OM Hospital, תצוגת הדפסה נקייה.

בנוסף: יומן ביקורת (Audit Log), ציר זמן למפגש, ארכיון שחרורים, וייצוא/ייבוא/איפוס נתונים.

## מבנה

- `index.html` — כל האפליקציה (CSS + HTML + JS, vanilla, ללא תלויות build).
- `manifest.webmanifest`, `sw.js`, `icon-192.png`, `icon-512.png` — מעטפת ה-PWA.
