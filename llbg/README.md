# נתב"ג ATC — סימולטור מגדל פיקוח ומכ"ם גישה (LLBG) מבוסס בינה מלאכותית

אפליקציית **Full-Stack** מלאה: שרת **Python / FastAPI** שמריץ את מנוע הפיזיקה
התעופתית ומגשר ל-**LLM** (OpenAI GPT-4o או Anthropic Claude), ולקוח **React**
שמצייר מסך מכ"ם בסגנון Eurocat/TopSky ומאפשר שיחת רדיו קולית חופשית בעברית עם
טייסי ה-AI.

> ✅ הכול נבדק: מנוע הפיזיקה, הפרסר העברי, חיבור ה-WebSocket ומחזור פקודה מלא
> (פקח → טייס → ביצוע) עובדים. **גם בלי מפתח API** המערכת פועלת במצב "טייס לא
> מקוון" (offline) עם פרסר עברי מובנה.

## ארכיטקטורה במבט-על

```
┌─────────────────────────┐         WebSocket (ws://…:8000/ws)        ┌──────────────────────────┐
│   דפדפן — React (Vite)   │  ───────────────────────────────────────▶│  שרת Python — FastAPI     │
│  BenGurionATC.jsx        │   state כל שנייה  ◀───────────────────────│  main.py  (לולאת 1 שנייה) │
│  • לולאת מכ"ם על Canvas  │   transmit (טקסט פקח)  ──────────────────▶│  sim.py   (פיזיקה תעופתית)│
│  • PTT (זיהוי דיבור he)  │   comm (תשובת טייס)  ◀────────────────────│  llm.py   (גשר ל-AI)      │
│  • הקראה קולית (TTS)     │                                          └────────────┬─────────────┘
└─────────────────────────┘                                                       │  HTTPS
                                                                   ┌───────────────▼──────────────┐
                                                                   │  OpenAI GPT-4o / Claude 3.5   │
                                                                   └──────────────────────────────┘
```

**זרימת פקודה:** הפקח לוחץ רווח ומדבר (או מקליד) → הטקסט נשלח בשרת → `llm.py`
שולח ל-AI את ה-System Prompt + מצב המטוס → ה-AI מחזיר JSON עם `radio` (התשובה
הקולית בעברית) ו-`commands` (כיוון/גובה/מהירות/פעולה) → השרת מבצע את הפקודה על
המטוס ומשדר את תשובת הטייס בחזרה לדפדפן, שמקריא אותה בקול עם אפקט "קששש" של רדיו.

```
llbg/
├── backend/
│   ├── main.py            # שרת FastAPI: WebSocket, לולאת סימולציה, ניתוב
│   ├── sim.py             # מנוע הפיזיקה: מטוסים, פניות, ILS, הפרדה, MSAW
│   ├── llm.py             # גשר ל-OpenAI/Anthropic + טייס offline בעברית
│   ├── requirements.txt
│   └── .env.example       # כאן שמים את מפתח ה-API
└── frontend/
    ├── index.html         # RTL + Tailwind (CDN) + פונט Heebo
    ├── src/BenGurionATC.jsx   # הקומפוננטה הראשית (מכ"ם + קשר + קול)
    ├── src/main.jsx
    ├── package.json
    └── vite.config.js
```

## מדריך התקנה והרצה — שלב אחר שלב

### דרישות מוקדמות (להתקין פעם אחת)
1. **Python 3.10+** — מ-[python.org](https://www.python.org/downloads/). בהתקנה
   ב-Windows סמנו "Add Python to PATH".
2. **Node.js 18+** — מ-[nodejs.org](https://nodejs.org/) (גרסת LTS).
3. דפדפן **Google Chrome** (חשוב: זיהוי הדיבור בעברית `he-IL` עובד הכי טוב ב-Chrome).

בדיקה שהכול הותקן (בטרמינל / שורת פקודה):
```bash
python --version      # או python3 --version
node --version
```

### שלב 1 — הרצת השרת (Backend)

פתחו טרמינל בתיקייה `llbg/backend`:

```bash
cd llbg/backend

# (מומלץ) סביבה וירטואלית
python -m venv .venv
# הפעלה:  ב-macOS/Linux:
source .venv/bin/activate
#         ב-Windows:
#         .venv\Scripts\activate

# התקנת התלויות
pip install -r requirements.txt
```

**הזנת מפתח ה-API שלכם:**
```bash
cp .env.example .env        # ב-Windows: copy .env.example .env
```
פתחו את הקובץ `.env` בעורך טקסט והדביקו מפתח **אחד**:

- ל-OpenAI:
  ```
  LLM_PROVIDER=openai
  OPENAI_API_KEY=sk-...המפתח-שלכם...
  ```
- ל-Claude (Anthropic):
  ```
  LLM_PROVIDER=anthropic
  ANTHROPIC_API_KEY=sk-ant-...המפתח-שלכם...
  ```
- **בלי מפתח כלל:** השאירו `LLM_PROVIDER=auto` — המערכת תעבוד עם הטייס ה-offline.

הרצת השרת:
```bash
uvicorn main:app --reload --port 8000
```
לבדיקה, פתחו בדפדפן `http://localhost:8000/` — אמורים לראות
`{"status":"ok", "llm":"openai"}` (או `anthropic`/`offline`).

### שלב 2 — הרצת הממשק (Frontend)

פתחו טרמינל **נוסף** (השאירו את השרת רץ) בתיקייה `llbg/frontend`:

```bash
cd llbg/frontend
npm install
npm run dev
```
פתחו בדפדפן את הכתובת שמודפסת (בדרך כלל **http://localhost:5173**).

> אם השרת רץ על מחשב/פורט אחר — ערכו את שורת `window.__LLBG_WS__` בקובץ
> `frontend/index.html`.

## איך משחקים

1. **בחירת מטוס:** לחצו על רצועת טיסה בצד או על מטרה במכ"ם (מסגרת לבנה תופיע סביבה).
2. **דיבור (PTT):** החזיקו את מקש **רווח** (Space) ודברו בעברית, למשל
   *"אל על 001, פנה ימינה לכיוון 280, רד וייצב שלושת אלפים"*. שחררו את הרווח —
   התשדורת נשלחת לטייס.
   - אפשר גם להקליד בתיבת הטקסט וללחוץ **שדר** / Enter, או ללחוץ על כפתורי הביטויים המהירים.
3. **הטייס עונה:** תשובת ה-AI מודפסת ביומן הקשר (בירוק) ומוקראת בקול עם צליל רדיו.
4. **נחיתה ידנית מלאה:** מטוס לא ינחת לבד! יש לאשר *"מאושר לגישת ILS מסלול 30"*,
   ואז *"רשאי לנחות מסלול 30"*. אם תשכחו — הטייס יבצע **הליכה סביב** מיוזמתו.
5. **המראות:** *"היערך והמתן"* → *"רשאי להמריא מסלול 26"*.
6. שמרו על **הפרדה** (3 מייל / 1,000 רגל) ועל **MSAW** (אל תנמיכו מתחת לגובה מינימלי).

### דוגמאות לביטויים שהמערכת מבינה (גם offline)
| בעברית | פעולה |
|---|---|
| `פנה ימינה/שמאלה לכיוון 280` | שינוי כיוון |
| `רד וייצב 3000` / `טפס וייצב 12000` | שינוי גובה |
| `האט למהירות 180` / `מהירות 210 קשר` | שינוי מהירות |
| `מאושר לגישת ILS מסלול 30 מהירות 180` | גישת ILS + הגבלת מהירות |
| `רשאי לנחות מסלול 30` | אישור נחיתה |
| `בצע הליכה סביב` | הליכה סביב |
| `היערך והמתן מסלול 26` / `רשאי להמריא מסלול 26` | המראה |
| `הסע לשער` | פינוי מסלול אחרי נחיתה |

> במצב המקוון (עם מפתח API) הטייס מבין **שפה חופשית** לחלוטין, מביע חשש, מתווכח
> ומגיב למצבי חירום — לא רק לביטויים שבטבלה.

## פתרון תקלות

- **אין קול / אין זיהוי דיבור:** השתמשו ב-Chrome, ואשרו לדפדפן גישה למיקרופון.
  זיהוי הדיבור של ה-Web Speech API נתמך בעיקר ב-Chrome.
- **`מנותק` במסך:** ודאו שהשרת (uvicorn) רץ על פורט 8000 ושלא חסום ע"י חומת אש.
- **שגיאת LLM / מכסה:** המערכת תיפול אוטומטית לטייס ה-offline וההודעה תירשם בלוג השרת.
- **CORS:** השרת כבר פתוח ל-CORS מכל מקור לצורכי פיתוח מקומי.

## הערות הנדסיות
- מנוע הפיזיקה רץ בשרת ב-`asyncio` כל שנייה; הדפדפן מבצע *dead-reckoning* קל בין
  עדכונים כדי שהתנועה תיראה חלקה.
- חילוץ הפקודות מה-AI נעשה ב-**JSON מובנה** (לא ניתוח טקסט שביר) — אמין ועמיד.
- מיקומי נקודות הדיווח (SUGGI/SOLIN/VELOX/PASAL) והמסלולים הם סכמטיים לצורכי
  אימון; ניתן להחליפם בקואורדינטות אמיתיות ב-`sim.py`.
